00 — 项目介绍

这是一个什么样的项目

在不读代码的情况下，用 3 分钟理解 WKEA-API 的全貌。
一句话定位
WKEA-API 是一个面向企业 B2B 采购与销售 场景的电商后台系统，覆盖商品、订单、合同、仓储、对账、供应商等核心业务流程。技术栈采用 Spring Boot 2.5.9 + MyBatis + MySQL + Redis + Elasticsearch，对外提供 PC、H5、APP、Vendor 四个端的 API。
核心业务模块
  • 商品中心：SPU/SKU、规格、属性、品牌、分类
  • 订单链路：销售单、采购单、报价单、销售合同、拼单
  • 需求与报价：需求单、报价单、自动化任务
  • 仓储与库存：出入库、货架、商品库存、调拨
  • 对账与发票：毛利、毛利更新、发票、应收回款
  • 客户与供应商：档案、地址、银行、配送
  • 在线客服：会话、消息、AI 集成（ChatGPT / DeepSeek）
  • 微信公众号：消息模板、用户绑定、菜单
  • 动态导出：通用 Excel / Word 模板导出（Apache POI）
  • 自动化工作流：任务、步骤、日志
技术栈
  • Java 11 + Spring Boot 2.5.9（Web、AOP、Validation）
  • MyBatis + MySQL + Druid 数据源
  • Redis（缓存、分布式锁、限流）
  • Elasticsearch 7.x（商品/订单搜索）
  • 腾讯云 COS（文件存储）、STS 临时凭证
  • 企业微信（消息推送、用户同步）
  • 微信支付、支付宝支付（异步回调）
  • 顺丰、快递 100 物流查询
  • Apache POI + POI-TL（Excel/Word 模板导出）
  • Shiro（权限认证）、Swagger（接口文档）
API 端划分
  • pc — PC 前台端（用户、订单、商品浏览）
  • app — 移动 App 端
  • vendor — 供应商端（报价、订单管理）
  • manage / managev2 — 管理后台 v1 与 v2（运营人员）
  • test — 测试与工具端（含 WeDoc、企业微信回调）

01 — 审计总结

整体评估

先给结论，再看细节。

WKEA-API 是 2014 年开始的 B2B 电商后台系统。9,675 次提交、73 位贡献者说明团队规模大、迭代活跃，但架构纪律跟不上业务速度。

P0 安全漏洞：腾讯云 COS 密钥以明文硬编码在 CosEcController.java:39 并提交到 Git 仓库。
